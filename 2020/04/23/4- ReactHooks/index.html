title: React Hooks 的一知半解

为什么会出现React Hooks ?

1.在React Class版本中，会有providers\consumeres\HOC 等其他抽象层组成的组件，会形成“嵌套地狱”。
（服用状态逻辑很难 - 在React Hooks中 ，可以写自定义hooks ）

2.在React Class中，有很多生命周期。在业务场景复杂的时候，会将不相关的代码放在同一个生命周期中。 会容易产生bug。
（复杂组件的生命周期 - 在React Hooks中, 通常使用Effect Hook 来对不同的业务代码进行处理 ）

React Hooks 各个API的用法？

useState (State Hook)

1- 是一个在函数调用时保存变量的方式， 与class 中的this.state的功能相同，
在函数推出后，变量会消失，而state中的变量会被React保留。

2- state只在组件首次渲染的时候被创建。在下一次重新渲染的时候，useState返回给我们当前的state.

3- state变量可以很好的存储对象和数组，可以将相关数据分为一组。

4- useState 支持函数式更新，即在setState的时候，可以返回一个prevState为参数的回调函数.
(useState 不会自动合并更新对象，需要手动返回)

5- 惰性初始state
初始state如果需要经过一个复杂的计算获得，可以通过函数调用返回state，从而初始state

6- 跳过state更新
调用State Hoo的更新函数饼传入当前的state的时候，React将跳过自组件和effect的执行。
如果渲染期间执行了高开销的计算，可以使用useMemo来优化

useEffect （Effect Hook 副作用函数）

useEffect 是把各个生命周期合并为了一个API，我们可以通过useEffect来写生命周期函数。
由于副作用函数是在组件内生命对，所以它们可以访问到组件的props与state.
🌟 React 会在每次渲染（执行DOM更新）后调用副作用函数 – 包括第一次渲染的时候。
🌟 多个effect生命的顺序依次调用组件的每个effect

如何清除副作用？

在副作用函数中返回一个函数来指定清除。

为什么在组件内部调用useEffect？

因为Hooks 使用了闭包机制，将state和props中保存在函数作用域中。
组件内部调用了useEffect，那么可以在effct中直接访问state/props变量。

如何对effect进行优化？

🌟通过“依赖项”跳过React对effect的调用
在组件重新渲染对时候，会比对依赖项的值，如果都是相等的，那么就会跳过这个effect。
这相当于一个 componenetDidUpdate的生命周期，来实现性能的优化

useCallback是做什么的？

useCallBack相当于一个优化性能的API，在依赖不变的情况下 (在我们的例子中是 count )，它会返回相同的引用，避免子组件进行无意义的重复渲染。
示例：https://segmentfault.com/a/1190000020108840

其实这3个API都是用于性能优化，

useRef的作用？

useRef 用于保存可变值，返回一个可变的ref对象（useRef 会在每次渲染时返回同一个 ref 对象），故可以使用它来存储ref节点， 也可以用于存储上一次的操作。
